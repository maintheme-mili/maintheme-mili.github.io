HTTP协议

发表于 2019-03-24 | 分类于 工作

HTTP的特性?

HTTP是无连接无状态的，构建在TCP/IP协议之上的应用层协议，默认端口80。

请求报文：状态行、请求头 、消息主体。

响应报文：状态行、响应头、响应正文。

条件GET?

HTTP条件GET为了减少多次访问同一页面造成的带宽浪费的解决方案。

机制：客户端发送请求询问服务器是否访问过此页面，如果访问过，就调用本地缓存与服务器数据做对比，如果没发送改变直接使用本地存储，反之正常返回更新的数据。

持久连接？

Keep-Alive通过向请求头中添加Connection: Keep-Alive来保证当前TCP连接，避免了重新 建立连接。长连接如何判断本次传输结束？一般通过判断Content-Length指示的大小来确定。

会话跟踪？

会话：从客户端向服务器发送请求并得到响应的全过程。

会话跟踪：监视会话的过程。HTTP协议无状态，无法保存客户信息，需要通过会话跟踪来监视客户的信息，例如：判断第二次登陆的是否为同一个用户。常见的会话跟踪：cookie、URL后缀、session

CSRF攻击？

伪造请求，冒充用户进行站内操作。一般通过Token对参数进行加密来抵御CSRF攻击。

HTTPS?

通过加密信道进行http内容传输协议。

TCP?

一种面向连接的可靠的字节流协议，通过校验、确认重传来保证可靠传输，只有两方彼此通信，不适用多播和广播。

UDP?

一种简单的传输层协议，不安全，传输数据报有长度，无连接，支持多播广播。(与TCP相反)

IP?

一种简单的网络层协议，提供点到点的服务。
